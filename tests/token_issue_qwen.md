## 🔍 針對你的三個問題逐一回覆：

### a. `handoff.json` 是做什麼的？

`handoff.json` 是一個**斷點續傳的狀態交接文件**，核心作用如下：

| 功能        | 說明                                                                                      |
| ----------- | ----------------------------------------------------------------------------------------- |
| 📋 記錄進度 | 儲存已處理的檔案清單、當前 Wave 編號、最後成功時間戳                                      |
| 🔄 狀態交接 | 當 Token 快耗盡時，Agent 將「當前狀態」寫入此文件，主動提示「請開新對話並載入此文件繼續」 |
| 🧠 減輕負擔 | 新對話只需讀取 `handoff.json` 的摘要狀態，不用繼承龐大的歷史對話，大幅節省 Token          |
| 🛡️ 防丟失   | 即使中途中斷，下次可從最後一個 Checkpoint 繼續，不用重頭來過                              |

📄 範例結構：

```json
{
  "project_root": "z:\\01",
  "last_completed_wave": 2,
  "processed_files": ["inv_001.jpg", "inv_002.pdf", ...],
  "summary": {"total": 50, "done": 18, "failed": 0},
  "next_action": "continue_wave_3",
  "timestamp": "2026-05-17T14:30:00Z"
}
```

### b. Context 限制 / IDE 外掛記憶體異常：如何解決？

這是「預防 + 應急」的組合策略：

#### ✅ 預防措施（從源頭避免爆量）

| 方法        | 實作方式                                                      | 效果                          |
| ----------- | ------------------------------------------------------------- | ----------------------------- |
| 📦 批次處理 | 每 10~15 個憑證為一個 Wave，處理完即輸出摘要並暫停            | 控制單次對話的 Token 成長曲線 |
| 🗑️ 精簡輸出 | 對話中只輸出「統計摘要」，詳細明細寫入 `.json` 檔案           | 減少 70%+ 的對話 Token 消耗   |
| 🧹 主動清理 | 每完成一個 Wave，Agent 自動建議執行 `/clear` 或 `/reset` 指令 | 手動釋放上下文壓力            |

#### 🆘 應急措施（異常發生時快速恢復）

| 情境                         | 對策                                                               |
| ---------------------------- | ------------------------------------------------------------------ |
| ⚠️ 出現 `token??` 或回應變慢 | 立即停止當前任務 → 執行 `/clear` → 載入最新 `handoff.json` 續跑    |
| 💥 IDE 外掛無回應            | 關閉並重開 IDE（會強制重置 Context）→ 讀取 `handoff.json` 恢復進度 |
| 🗑️ 檔案被誤刪                | 從 `handoff.json` 的 `processed_files` 清單比對，只重跑缺失部分    |

#### 🔧 技術層面優化（開發者可實作）

```python
# 範例：Agent 在每批次結束時自動生成 handoff
def save_checkpoint(wave_id, processed, summary):
    handoff = {
        "wave": wave_id,
        "done": processed,
        "summary": summary,
        "next_prompt": f"請從 wave_{wave_id+1} 繼續，已忽略歷史詳細日誌"
    }
    with open("handoff.json", "w", encoding="utf-8") as f:
        json.dump(handoff, f, ensure_ascii=False, indent=2)
```
